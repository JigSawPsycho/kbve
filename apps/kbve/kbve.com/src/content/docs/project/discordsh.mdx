## DiscordSH

This is unofficial notes and docs for discordsh.


### WebSockets

For testing the websockets, we have these two JSONs

Nested Direct Command Set / Get:

```json

{
  "message": {
    "Command": {
      "command": {
        "Set": { "key": "1", "value": "pop", "ttl": 30 }
      }
    }
  }
}

```

Then to get:

```json

{
  "message": {
    "Command": {
      "command": {
        "Get": { "key": "1" }
      }
    }
  }
}

```

Watch Command:

```json

{
  "message": {
    "Watch": {
      "key": "1"
    }
  }
}

```

Thin JSON 

```json

{ "type": "set", "key": "mykey", "value": "myvalue", "ttl": 30 }

```

Thin JSON 2 

```json

{ "type": "set", "key": "mykey", "value": "myvalue", "ttl": 30 }


```

Thin Watch 

```json

{ "type": "watch", "key": "mykey" }
```
